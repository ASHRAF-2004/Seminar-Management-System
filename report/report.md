## Introduction
This lab exercise implements a Java Swing based Seminar Management System that supports students (presenters), evaluators, and coordinators. The objective is to deliver a coherent OO design that aligns UML artifacts with a runnable application, offering clean role-based dashboards, validations, and simple persistence via CSV files.

## Requirements / Features Summary
- **Authentication / Role Selection:** Login by role (Student, Evaluator, Coordinator) with demo credentials.
- **Student Module:** Browse seminars, enroll/register, view enrollment status.
- **Coordinator Module:** Create and update seminar details, monitor participant enrollments.
- **Evaluator Module:** Record rubric scores (problem clarity, methodology, results, presentation) and comments per seminar.
- **Seminar Listing & Details:** Table-based listings with dates, venues, and status.
- **Persistence:** CSV storage under `data/` for users, seminars, enrollments, and evaluations.
- **Validation & Feedback:** Required field checks, informative dialogs, and average score display.

## System Design
- **Use Case Diagram:** Captures the interactions of Student, Evaluator, and Coordinator with core use cases: login, browse, enroll, manage seminars, evaluate presentations, and monitor participants.
- **Class Diagram:** Organized packages (`model`, `repository`, `service`, `ui`, `util`). Key classes include `Seminar`, `Enrollment`, `Evaluation`, `User`, and supporting enums. Services encapsulate business rules and delegate to repositories. UI frames depend on services only, ensuring separation of concerns.
- **Sequence Diagrams:**
  1. Student login and seminar browsing.
  2. Student enrollment flow with persistence.
  3. Coordinator creating/updating seminars.
  4. Evaluator recording/overwriting evaluations and viewing averages.

## Implementation
- **Architecture:** MVC-inspired layering: Swing UI frames -> services -> repositories -> CSV files. IDs generated by `IdGenerator`.
- **UI Screens:**
  - `RoleSelectionFrame` provides role choice and credential entry.
  - `StudentDashboard` lists seminars and allows enrollment with status tracking.
  - `CoordinatorDashboard` offers create/edit seminar dialog and participant table.
  - `EvaluatorDashboard` lets evaluators score seminars using rubric spinners and comments.
- **Persistence:** Simple CSV per entity using `FileUtils`; repositories preload sample users and seminars for immediate demo use.
- **Extensibility Notes:** New roles or statuses can be added via enums and additional dashboards; repositories abstract storage to allow migration to databases.

## Testing / Sample Runs
- Verified launching via `java ui.Main` (after compilation) to navigate each dashboard.
- Sample credential set (seeded): `stu1/pass`, `eval1/pass`, `coord1/pass`.
- Enrollment and evaluation actions persist to CSV and are reloaded on restart.
